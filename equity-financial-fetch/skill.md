# Equity Financial Fetch (财报数据获取)

## 核心定位

本 skill 用于获取上市公司的财报原始数据，不做深度分析。作为其他分析 skill（如 equity-initial-analysis、equity-earnings-analysis）的数据获取基础，也可独立使用。

### 核心原则

1. **数据优先**：专注于获取准确、完整的财报原始数据
2. **来源透明**：所有数据必须标注来源，确保可追溯
3. **多源备份**：当首选数据源不可用时，自动切换到备选数据源
4. **格式灵活**：支持多种输出格式，满足不同使用场景

---

## 适用场景

- 获取指定上市公司的财报原始数据
- 查询特定财务指标
- 导出财务数据表格（CSV）
- 为其他分析 skill 提供数据支撑
- 跨市场财报数据获取（美股、A股、港股）

## 默认核心指标

当用户未指定具体指标时，默认获取以下核心财务指标：

### 收入与利润指标

| 指标名称 | 说明 | 单位 |
|----------|------|------|
| 营业收入 | 公司主营业务收入 | 亿元 |
| 营业收入同比 | 营业收入同比增长率 | % |
| 归母净利润 | 归属于母公司股东的净利润 | 亿元 |
| 归母净利润同比 | 归母净利润同比增长率 | % |

### 盈利能力指标

| 指标名称 | 说明 | 单位 |
|----------|------|------|
| 毛利率 | 毛利润/营业收入 | % |
| 营业利润率 | 营业利润/营业收入 | % |
| 净利率 | 净利润/营业收入 | % |
| ROE | 净资产收益率 = 净利润/股东权益 | % |
| ROA | 总资产收益率 = 净利润/总资产 | % |

### 现金流指标

| 指标名称 | 说明 | 单位 |
|----------|------|------|
| 净现比 | 经营现金流/净利润 | 倍 |
| 经营现金流净额 | 经营活动产生的现金流量净额 | 亿元 |

### 资本配置指标

| 指标名称 | 说明 | 单位 |
|----------|------|------|
| 资本开支 | 资本性支出（CapEx） | 亿元 |
| 自由现金流 | 经营现金流 - 资本开支 | 亿元 |

### 股东回报指标

| 指标名称 | 说明 | 单位 |
|----------|------|------|
| 股息支付金额 | 现金分红总额 | 亿元 |
| 股票回购金额 | 股票回购总额 | 亿元 |

## 完整财务报表

当用户需要完整报表时，可获取以下三张表：

### 资产负债表（Balance Sheet）

主要包含：
- 资产总额、流动资产、非流动资产
- 负债总额、流动负债、非流动负债
- 股东权益
- 现金及现金等价物
- 有息负债等

### 利润表（Income Statement）

主要包含：
- 营业收入
- 营业成本
- 毛利润
- 营业利润
- 净利润
- 每股收益等

### 现金流量表（Cash Flow Statement）

主要包含：
- 经营活动现金流
- 投资活动现金流
- 融资活动现金流
- 自由现金流等

## API 调用接口

本 skill 支持被其他 skill 调用，提供标准化的数据接口。

### 输入参数

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| ticker | string | 是 | 股票代码 |
| market | string | 否 | 市场类型（us/a/hk），默认自动识别 |
| years | int | 否 | 获取年数，默认10 |
| data_type | string | 否 | 数据类型（core/balance/income/cashflow/all），默认core |
| output_format | string | 否 | 输出格式（markdown/csv/both），默认both |
| output_dir | string | 否 | 输出目录，默认~/financial-data/ |

### 返回数据结构

```python
{
    "ticker": "AAPL",
    "company_name": "Apple Inc.",
    "market": "us",
    "data_source": "SEC API",
    "data_cutoff_date": "2024-12-31",
    "fetch_date": "2026-03-30",
    "data": {
        "years": [2024, 2023, 2022, 2021, 2020],
        "indicators": {
            "revenue": {"2024": 1000, "2023": 900, ...},
            "net_income": {"2024": 200, "2023": 180, ...},
            ...
        }
    },
    "output_files": {
        "markdown": "/path/to/output.md",
        "csv": "/path/to/output.csv"
    }
}
```

### 调用示例

```python
# 从 equity-initial-analysis 调用
result = equity_financial_fetch(
    ticker="AAPL",
    market="us",
    years=10,
    data_type="core",
    output_format="both"
)

# 使用返回数据
revenue_data = result["data"]["indicators"]["revenue"]
```

## 市场识别规则

### 美股识别

- 股票代码为纯字母：AAPL、MSFT、GOOGL
- 股票代码包含 . 后缀：BRK.B、GOOG

### A股识别

- 股票代码为6位数字，以 0、3、6 开头：
  - 000001（平安银行）
  - 600519（贵州茅台）
  - 300750（宁德时代）

### 港股识别

- 股票代码为4-5位数字，以 0、1、2 开头：
  - 00700 或 0700（腾讯控股）
  - 09988 或 9988（阿里巴巴-SW）
- 或明确指定市场为港股

## 错误处理

### 常见错误及处理方式

| 错误类型 | 说明 | 处理方式 |
|----------|------|----------|
| 股票代码不存在 | 输入的股票代码无法识别 | 提示用户检查股票代码 |
| 数据源不可用 | 首选数据源无法访问 | 自动切换到备选数据源 |
| 数据不完整 | 部分指标数据缺失 | 标注缺失数据，返回可用数据 |
| 市场不支持 | 股票所属市场暂不支持 | 提示用户当前支持的市场范围 |

### 错误信息格式

```markdown
**获取失败**

错误类型：{错误类型}
错误说明：{详细说明}
建议操作：{用户可采取的操作}
```

## 注意事项

1. **数据延迟**：财报数据通常在季度结束后几周内可用
2. **市场差异**：不同市场的财报披露规则不同，数据格式可能有差异
3. **货币单位**：
   - 美股：默认亿美元
   - A股：默认亿元
   - 港股：默认亿港元
4. **数据准确性**：所有数据仅供参考，投资决策请以官方财报为准
5. **更新频率**：建议定期更新数据，确保使用最新财报数据
